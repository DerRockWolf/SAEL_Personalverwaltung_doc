    Source Code¶

    Der Source Code kann entweder direkt auf GitHub gelesen werden,
    oder lokal, nachdem der Source Code heruntergeladen wurde:

    Der Source Code kann entweder mit 

    git clone https://github.com/DerRockWolf/SAEL_Project_Personalverwaltung.git
    geklont werden oder über die Option Download ZIP (Direct Download) als ZIP Archiv heruntergeladen werden.

    Build¶

    Da wir in unserem Projekt einen Teil des System.Data.SQLite NuGet-Paket verwenden muss dies zuerst geladen werden:
    Dies geschied automatisch beim ersten Build.
    Es ist aber auch möglich das Paket zuvor wiederherzustellen:

    1. Rechtsklick auf die Projektmappe (Solution) im Projektmappen-Explorer
    2. Linksklick auf NuGet-Pakete wiederherstellen

    Danach kann der Source Code gelesen werden und die gelben Warndreiecke vor den Verweisen im Projektmappen-Explorer verschwinden.

    WICHTIG

    Da Visual Studio (MSBuild.exe) die benötigten sqlite.interop.dll Dateien beim 1. Build nicht automatisch in den Build Pfad kopiert muss wie folgt vorgegangen werden:

    1. Projektmappe bereinigen: Erstellen > Projektmappe bereinigen
    2. Projektmappe erstellen: Erstellen > Projektmappe erstellen

    Features¶

    Datenspeicherung in SQLite Datenbank

    • Erstellen neuer Datenbank-Dateien
    • Öffnen von bestehenden Datenbank-Dateien
      • Überprüfung des Datenbank Schemas

    Veränderung der Daten

    • Erstellung neuer Einträge
    • Bearbeitung von bestehenden Einträgen
    • Löschen von bestehenden Einträgen
    • Verhalten
      • Pflichfelder (gekennzeichnet mit *) [Vorname, Nachname, Adresse, Telefonnummer]
      • Überprüfung der Eingaben (Erlaubte Zeichen siehe Tooltip auf Label und Textboxen)

    Darstellung

    • Einträge werden in DataGridView dargestellt
    • Anzahl der Einträge wird angezeigt
    • Anzahl der angezeigten Einträge anpassbar
    • Alphabetische Sortierung (Nachname, Vorname) möglich
    • Suchen (Felder spezifisch)
      • Suche über mehrere Spalten möglich
      • Suche mit Wildcards/Auslassungszeichen (*) möglich

    Misc

    • Speicherung und automatisches Öffnen der zuletzt verwendeten Datenbank
    • Darstellung des Pfads im Forms Titel
    • Wechseln zum Startbildschirm

    Datenbank Schema¶

    Damit bereits bestehende Datenbankdateien verwendet werden können muss die Tabelle in personalverwaltung umbenannt werden.
    Die Tabelle muss folgende Spalten beinhalten:
    first_name last_name address number email position entry retirement salary
    Die Spalten müssen Typ TEXT sein und die Option Not null aktiviert haben.
